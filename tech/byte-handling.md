# 바이트 처리

HANPA는 HWP/HWPX 문서를 열고 저장할 때 이진 데이터(Byte)를 직접 다루는 부분이 중요합니다.

## 파일 입출력

- Electron 메인 프로세스는 Node.js `fs`를 통해 파일을 읽고 씁니다.
- 렌더러는 `ArrayBuffer`, `Uint8Array` 같은 바이너리 버퍼를 사용해 문서를 전달받고 처리합니다.

## HWP/HWPX 데이터

- HWP는 바이너리 형식이며, HWPX는 XML 기반이지만 압축/바이너리 데이터가 포함될 수 있습니다.
- `@rhwp/core`는 이러한 문서를 파싱하고, 내부 모델을 구성한 뒤 다시 시리얼라이즈합니다.

## 바이트와 문자열

- 바이너리 데이터를 문자열로 변환할 때는 인코딩 오류를 주의해야 합니다.
- 텍스트 데이터는 `UTF-8`을 기준으로 하며, HWP 내부 문자 처리도 엔진이 담당합니다.

## 메모리와 성능

- 큰 문서를 다룰 때는 불필요한 복사를 최소화해야 합니다.
- `Uint8Array` 뷰를 사용해 버퍼를 직접 전달하고, 필요한 경우 부분 버퍼를 참조합니다.

## HANPA에서의 처리 흐름

1. 사용자 파일 열기 요청
2. 메인 프로세스가 `fs.readFile`로 바이트를 읽음
3. 바이트 버퍼를 렌더러에 전달
4. `@rhwp/core`가 파싱하여 문서 모델로 변환
5. 편집 후 저장 요청 시 다시 바이트로 시리얼라이즈
6. 메인 프로세스가 파일로 기록
