# TypeScript

HANPA는 TypeScript를 사용하여 코드 안정성을 높이고 개발 생산성을 개선합니다.

## 이유

- 타입 시스템을 통해 런타임 오류를 예방
- 명확한 인터페이스와 데이터 모델 정의
- 코드 완성 및 리팩터링 지원

## 구성

- `tsconfig.json`에 컴파일 옵션과 경로 설정이 포함되어 있습니다.
- `src/`의 대부분 파일은 `*.ts` 또는 `*.tsx` 형식으로 작성됩니다.

## 장점

- `@rhwp/core`와 같은 엔진 인터페이스 사용 시 타입 안정성 확보
- 컴파일 단계에서 타입 오류를 발견하여 버그 감소
- 코드 유지보수성과 가독성 향상

## HANPA 구현에서의 활용

- DOM, Electron IPC, HWP 문서 모델 등의 타입을 정의
- 명령 처리, 상태 관리, 렌더링 로직에서 타입 기반 검증
- `src/`와 `third_party/rhwp/` 간 상호작용에서 계약(Contract)을 명시

## 권장 방식

- 가능한 경우 `any` 대신 구체적인 타입, 인터페이스, 유니온 타입 사용
- 모듈 간 경계에서는 타입 정의를 명확하게 유지
- 컴파일러 옵션으로 `strict` 모드를 켜면 안정성을 높일 수 있습니다
