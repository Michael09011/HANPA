# 기여 가이드

HANPA 프로젝트에 관심을 가져주셔서 감사합니다! 이 문서는 버그 리포트, 기능 제안, 코드 기여, 문서 개선 등 프로젝트에 기여하는 방법을 안내합니다.

## 기여 방법

### 1. 이슈 생성

- 버그를 발견하면 상세한 재현 단계, 기대 동작, 실제 동작을 포함해서 이슈를 등록하세요.
- 기능 제안은 어떤 문제를 해결하고 싶은지, 제안하는 변경 사항이 어떤 이점을 줄지 설명해 주세요.
- 가능한 경우 스크린샷, 로그, 샘플 파일 또는 재현 가능한 코드 조각을 첨부해 주세요.

### 2. 코드 기여

- 먼저 기존 이슈를 확인하고, 관련 이슈가 없는 경우 새로운 이슈를 생성한 뒤 작업을 시작하세요.
- 작업 내용이 확정되면 브랜치를 생성합니다.
  - 예: `feature/새기능-이름`, `fix/버그-이름`, `docs/문서-수정`
- 코드 스타일과 기존 패턴을 따라 작성해 주세요.
- 변경 사항이 크거나 영향 범위가 넓은 경우 PR에 충분한 설명을 작성해 주세요.

### 3. 테스트

- 가능한 경우 새로운 기능이나 버그 수정을 위한 테스트를 추가하거나 기존 테스트를 업데이트하세요.
- `e2e/` 디렉터리에는 엔드투엔드 테스트가 포함되어 있습니다.
- `src/` 내부 로직에 대한 단위 테스트가 있는 경우 프로젝트의 테스트 룰을 확인한 뒤 작성해 주세요.

### 4. 문서 개선

- 프로젝트 설명, 사용법, 설정 또는 개발자 문서를 개선하려면 README나 적절한 `.md` 파일을 수정해 주세요.
- 용어, 예제, 설치 방법 등을 명확하게 작성하면 다른 기여자에게 큰 도움이 됩니다.

### 5. PR 작성

- PR 제목은 간결하고 명확하게 작성합니다.
- PR 본문에 변경 내용, 관련 이슈, 테스트 방법을 포함합니다.
- 리뷰어가 변경 사항을 쉽게 이해할 수 있도록 요약 정보를 제공합니다.

## 커뮤니케이션

- 이 저장소의 이슈와 PR에서 소통합니다.
- 리뷰어의 의견을 존중하고 필요한 경우 질문을 남기세요.
- 코드 스타일과 컨벤션에 대해 확실하지 않은 경우 먼저 논의하는 것이 좋습니다.

## 코드 스타일

- TypeScript/JavaScript 코드는 기존 스타일을 따릅니다.
- CSS나 HTML 수정 시 프로젝트의 기존 규칙을 유지합니다.
- 명시적인 타입과 가독성을 우선합니다.

## 감사 인사

의견을 주거나 코드로 기여해 주시는 모든 분께 감사드립니다. HANPA가 더 나아질 수 있도록 함께 만들어가요!
